# 사이드바 컴포넌트 분리 및 반응형 기능 개선
### 0. Windsurf
- **SWE-1.5 모델 사용**

## 작업 개요
main-content 내부의 sidebar와 editor-container div를 별도 컴포넌트로 분리하고, 1024px 이하 화면에서 사이드바 숨김/표시 기능을 구현하여 EditorContainer의 공간 활용도를 개선

## 주요 변경사항

### 1. 컴포넌트 분리
- **Sidebar.tsx**: RulePanel, TemplatePanel, AiPanel을 포함하는 사이드바 컴포넌트
- **EditorContainer.tsx**: Input/Output EditorPanel과 정렬 버튼을 포함하는 에디터 컨테이너 컴포넌트
- **App.tsx**: main-content 구조 단순화 및 컴포넌트 조합

### 2. 반응형 사이드바 기능
- **1024px 이상**: 사이드바 항상 표시 (기존과 동일)
- **1024px 이하**: 사이드바 숨김/표시 토글 기능 활성화
- **토글 버튼**: 화면 크기에 따라 동적으로 위치 변경

### 3. 상태 관리 추가
```typescript
const [isSidebarHidden, setIsSidebarHidden] = useState(false)
const toggleSidebar = () => setIsSidebarHidden(!isSidebarHidden)
```

### 4. CSS 스타일 개선

#### 사이드바 스타일
- 기본 너비: 320px 유지
- 1024px 이하: 절대 위치로 변경, 슬라이드 애니메이션 적용
- 숨김 상태: `transform: translateX(-100%)`으로 왼쪽으로 슬라이드

#### 메인 컨텐츠 스타일
- 사이드바 숨김 시: `max-width: 100%`로 전체 너비 활용
- 상대 위치 추가: 토글 버튼 위치 기준

#### 토글 버튼 스타일
- 기본: 숨김 (`display: none`)
- 1024px 이하: 표시 (`display: flex`)
- 동적 위치: 사이드바 상태에 따라 left 값 조정
- 아이콘: `◀` (표시) / `▶` (숨김)

## 기능 상세

### 화면 크기별 동작
- **1024px 초과**: 사이드바 항상 표시, 토글 버튼 숨김
- **1024px 이하**: 
  - 사이드바 숨김 가능
  - 토글 버튼 표시
  - EditorContainer 전체 너비 사용

### 애니메이션 효과
- 사이드바: 0.3s ease-in-out 전환 효과
- 토글 버튼: 0.2s ease 호버/클릭 효과
- 부드러운 슬라이드 인/아웃

### 사용자 경험 개선
- 더 넓은 에디터 공간 활용 가능
- 직관적인 토글 버튼 위치 및 아이콘
- 기존 기능 완전 유지
