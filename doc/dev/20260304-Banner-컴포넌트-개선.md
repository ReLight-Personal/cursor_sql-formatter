## 주요 변경 사항

### 1. Arrow-icon 방향 로직 수정
**문제점**: Top Banner의 arrow-icon 방향이 직관적이지 않음
- 기존: 숨겨졌을 때 `arrow-up`, 보일 때 `arrow-down`
- 수정: 숨겨지지 않았을 때 `arrow-up`, 숨겨졌을 때 `arrow-down`

**개선 내용**:
- Top Banner가 위로 사라지므로, 보일 때는 위쪽 화살표(▲) 표시
- 숨겨졌을 때는 아래쪽 화살표(▼) 표시하여 다시 열 수 있음을 시각적으로 표현

### 2. 독립적인 Banner 제어
**문제점**: 상단과 하단 Banner가 동일한 state를 공유하여 하나를 숨기면 다른 것도 함께 숨겨짐

**개선 내용**:
- `isBannerHidden` → `isTopBannerHidden`, `isBottomBannerHidden`으로 state 분리
- `toggleBanner` → `toggleTopBanner`, `toggleBottomBanner`로 함수 분리
- 각 Banner가 독립적으로 숨기기/보이기 기능 수행

### 3. 숨겨진 Banner의 버튼 가시성 확보
**문제점**: Banner가 숨겨질 때 버튼도 함께 사라져 다시 표시할 수 없음

**개선 내용**:
- Banner 숨김 시 버튼을 `position: fixed`로 화면에 고정
- 상단 Banner: 화면 상단 우측에 버튼 표시
- 하단 Banner: 화면 하단 우측에 버튼 표시
- 버튼에 배경색, 테두리, 그림자 효과 추가하여 가시성 향상

### 4. 향후 좌우 배치 확장 준비
**확장성 고려**: 다음 단계에서 좌우 Banner 배치를 대비한 구조 개선

**개선 내용**:
- `BannerProps`의 `position` 타입 확장: `'top' | 'bottom' | 'left' | 'right'`
- Arrow 방향 로직에 좌우 방향 추가: `arrow-left`, `arrow-right`
- Aria-label 개선: 각 위치별 접근성 레이블 제공
- CSS 스타일 준비: 좌우 Banner용 레이아웃 및 애니메이션

## 기술적 상세

### Arrow 방향 규칙
| Position | 보일 때 | 숨겨졌을 때 | 설명 |
|----------|---------|-------------|------|
| Top | arrow-up (▲) | arrow-down (▼) | 위로 사라짐 |
| Bottom | arrow-down (▼) | arrow-up (▲) | 아래로 사라짐 |
| Left | arrow-left (←) | arrow-right (→) | 왼쪽으로 사라짐 |
| Right | arrow-right (→) | arrow-left (←) | 오른쪽으로 사라짐 |

### CSS 클래스 추가
```css
/* 좌우 Banner 레이아웃 */
.banner-left, .banner-right {
  width: 250px;
  flex-direction: column;
  height: auto;
  /* ... */
}

/* 숨김 애니메이션 */
.banner-left.banner-hidden { transform: translateX(-100%); }
.banner-right.banner-hidden { transform: translateX(100%); }

/* Arrow 방향 */
.arrow-left { transform: rotate(90deg); }
.arrow-right { transform: rotate(-90deg); }
```

### TypeScript 인터페이스
```typescript
interface BannerProps {
  isHidden?: boolean
  onToggleHide?: () => void
  position?: 'top' | 'bottom' | 'left' | 'right'
}
```
