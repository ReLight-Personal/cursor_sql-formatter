# 실시간 방언 감지 및 App 로직 정리
### 0. Claude
- **Sonnet 4.6 모델 사용**

## 개요

입력 영역(Input)에 SQL이 입력되거나 삭제될 때 포매팅 버튼 클릭 없이
방언(Dialect)이 즉시 감지되어 배지에 표시되도록 개선.
기존 App.tsx의 중복된 로직도 함께 정리.

---

## 1. 문제

이전 구현에서 `detectedDialect`는 `runFormat()` 내부에서만 갱신되었음.

```
[변경 전 흐름]
inputSql 입력
  → 방언 감지 없음 (배지 미표시)
  → "정렬하기" 버튼 클릭
  → runFormat() 실행
  → detectedDialect 갱신 → 배지 표시
```

즉, **버튼을 눌러야만** 배지가 나타나는 구조였음.

## 2. 변경 사항

### `src/utils/formatSql.ts`

**`detectDialectOnly()` 함수 신규 추가**

포매팅 없이 방언 감지만 수행하는 경량 함수.
입력 중 실시간 감지에 사용하여 포매팅 비용 없이 배지만 업데이트.

```typescript
export function detectDialectOnly(sql: string): { dialect: SqlDialect; label: string } {
  const formatter = new SqlFormatter()
  const dialect = formatter.detectDialect(sql)
  return { dialect, label: DIALECT_LABEL[dialect] }
}
```

**불필요한 import 정리**

- `formatSql` named import 제거 (내부에서 `SqlFormatter` 직접 사용으로 통일)

### `src/App.tsx`

#### 실시간 방언 감지 useEffect 추가

`inputSql`이 변경될 때마다 debounce(300ms) 후 `detectDialectOnly()` 호출.
포매팅과 완전히 독립적으로 동작하므로 입력 중에도 배지가 즉시 갱신됨.

```typescript
const DETECT_DEBOUNCE_MS = 300
const detectTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null)

useEffect(() => {
  if (detectTimerRef.current) clearTimeout(detectTimerRef.current)
  if (!inputSql.trim()) { setDetectedDialect(''); return }

  detectTimerRef.current = setTimeout(() => {
    const { label } = detectDialectOnly(inputSql)
    setDetectedDialect(label)
  }, DETECT_DEBOUNCE_MS)

  return () => { if (detectTimerRef.current) clearTimeout(detectTimerRef.current) }
}, [inputSql])
```

**debounce를 적용한 이유**
- 글자를 연속 입력할 때마다 매번 감지 함수를 호출하면 불필요한 연산 발생
- 300ms 딜레이로 입력이 잠시 멈췄을 때만 감지 실행

#### 중복 로직 제거 및 관심사 분리

기존에는 동일한 포매팅 로직이 `runFormat()`과 `useEffect([rules, customRules])` 두 곳에 중복되어 있었음.

| 구분 | 역할 | 트리거 |
|------|------|--------|
| `useEffect([rules, customRules])` | 규칙 변경 시 자동 재포매팅 | rules / customRules 변경 |
| `runFormat()` | 명시적 포매팅 실행 | "정렬하기" 버튼 클릭 |
| `useEffect([inputSql])` | 실시간 방언 감지만 수행 | inputSql 변경 |

→ 각 useEffect의 역할을 단일 책임으로 분리하여 의도가 명확해짐.

#### 레이아웃 토글 핸들러 개선

```typescript
// 변경 전
const toggleTopBanner = () => setIsTopBannerHidden(!isTopBannerHidden)

// 변경 후 — 클로저 오래된 값 참조 방지
const toggleTopBanner = () => setIsTopBannerHidden(v => !v)
```

## 3. 변경 후 흐름

```
[방언 감지]
inputSql 입력/삭제
  → useEffect([inputSql]) 트리거
  → debounce 300ms 대기
  → detectDialectOnly(inputSql) 실행
  → detectedDialect 갱신 → 배지 즉시 표시

[포매팅]
"정렬하기" 버튼 클릭
  → runFormat() 실행
  → formatWithRules() → setOutputSql()
  → detectedDialect도 함께 갱신 (포매팅 결과 기준 동기화)

[규칙 변경 시 자동 재포매팅]
rules / customRules 변경
  → useEffect([rules, customRules]) 트리거
  → formatWithRules() → setOutputSql() 자동 갱신
```
