# [2단계] 규칙 제어 패널 및 상태 관리 — 구현 요약

## 목표
- 정렬 규칙 On/Off 및 실시간 반영
- 콤마 위치(앞/뒤) 선택
- 각 규칙 적용 상태가 Output에 실시간 반영되도록 연동

---

## 구현 내용

### 1. 포매팅 규칙 타입 및 상태
- **파일**: `src/types/formatRules.ts`
- **타입**
  - `KeywordCase`: `'upper' | 'lower' | 'preserve'`
  - `CommaPosition`: `'leading' | 'trailing'`
  - `IndentType`: `'spaces' | 'tabs'`
  - `FormatRulesState`: 위 옵션 + 규칙별 On/Off 플래그
- **기본값**: `defaultFormatRules` (예: 예약어 대문자, 콤마 뒤, Space 2칸 등)

### 2. 규칙 제어 패널 UI
- **파일**: `src/components/RulePanel.tsx`, `RulePanel.css`
- **위치**: 메인 영역 왼쪽 사이드바
- **구성**
  - **예약어 대소문자**: 체크박스(On/Off) + 라디오(대문자 / 소문자 / 유지)
  - **콤마 위치**: 체크박스 + 라디오(앞 Leading / 뒤 Trailing)
  - **들여쓰기**: 체크박스 + Space / Tab 라디오, Space 시 2칸·4칸 선택
  - **연산자 공백**: 체크박스 + 공백 있음 / 공백 없음(Dense) 라디오

### 3. 규칙 적용 로직
- **파일**: `src/utils/formatSql.ts`
- **함수**: `formatWithRules(sql, rules)`
  - 활성화된 규칙만 반영해 `sql-formatter` 옵션 생성 (`tabWidth`, `useTabs`, `keywordCase`, `denseOperators`)
  - **콤마 Leading**: 라이브러리 기본은 Trailing이므로, Leading 선택 시 포맷 결과를 후처리로 줄 끝 `,` → 다음 줄 앞 `,` 로 변환

### 4. 실시간 Output 반영
- **파일**: `src/App.tsx`
- **레이아웃**: `Banner` 아래 `main-content` (RulePanel + editor-container)
- **상태**: `rules`를 `useState`로 관리, `RulePanel`에 `rules` / `onChange` 전달
- **동작**
  - **정렬하기** 클릭: 현재 Input + rules로 포맷 후 Output 갱신
  - **규칙 변경**: `useEffect`로 `rules` 변경 시에만 재포맷하여 Output 갱신 (Input 변경 시에는 버튼으로만 포맷)

## 확인 방법

1. `npm run dev` 실행 후 왼쪽 규칙 패널에서 각 항목 On/Off 및 라디오 변경
2. Input에 SQL 입력 후 **정렬하기** 클릭
3. 규칙(예: 대문자→소문자, 콤마 뒤→앞)을 바꾸면 Output이 자동으로 다시 포맷되는지 확인
