# [1단계] 기초 레이아웃 및 Core 포매팅 기능 — 구현 요약

## 목표
- 좌우 분할 에디터 UI 구성
- 기본 SQL 포매팅(들여쓰기, 대소문자 변환) 구현
- 상단 배너 공간(Placeholder) 확보

---

## 구현 내용

### 1. 프로젝트 초기 설정
- **React 18** + **TypeScript** + **Vite** 프로젝트 구성
- 의존성: `react`, `react-dom`, `sql-formatter`
- 개발 의존성: `@vitejs/plugin-react`, `typescript`, `vite`

### 2. 좌우 분할 텍스트 에디터 UI
- **Input 패널**: 원본 SQL 입력
- **Output 패널**: 정렬 결과 표시 (readOnly)
- 중앙 **정렬하기** 버튼으로 포맷 실행
- 반응형 레이아웃 (`editor-container` flex)

### 3. 기본 SQL 포매팅
- `sql-formatter` 라이브러리 사용
- 버튼 클릭 시:
  - **들여쓰기** 적용 (기본 2칸 스페이스)
  - **예약어 대소문자** 변환 (기본: 대문자)
- 포맷 실패 시 에러 메시지를 Output에 표시

### 4. 배너 영역
- 상단 **Banner** 컴포넌트
- 앱 타이틀 "SQL-Tailor", 부제 표시
- 추후 광고·공지용 **Placeholder** 영역 확보

| 경로 | 설명 |

## 실행 및 확인

```bash
npm install
npm run dev
```

브라우저에서 왼쪽에 SQL 입력 후 **정렬하기** 클릭 시 오른쪽에 포맷 결과가 표시되는지 확인하면 됩니다.
